## B. P0 安全契约修复

### B-1 main.py: dev_mode=False 必须 raise

`src/api/main.py` lifespan 中：

- `check_signing_key()` 返回 False 时
  - `dev_mode=True` 维持旧行为（`logger.warning`，方便本地排错）
  - `dev_mode=False` 直接 `raise RuntimeError`，lifespan 启动失败

否则生产配错/未配 Ed25519 私钥会无声启动，agent 拒收所有 rule_update 但
运维无感知——比"完全不校验"更危险，因为"启动成功"是谎言。

### B-2 signing.py: 调试日志 gating

`src/common/config/settings.py` 新增：

```python
agent_debug: bool = False  # V4.1 (P0-2): mirrors Go-side AGENT_DEBUG=1
```

`src/agents/signing.py:sign_message`：

```python
if get_settings().agent_debug:
    logger.info("instruction_signed_debug", canonical=sign_payload)
```

Py 侧默认不再把 raw payload 字段（hostnames / paths / cve ids）写入 INFO
journalctl。operator 排错时显式设 `AGENT_DEBUG=1` 即恢复。
