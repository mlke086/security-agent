一、项目概述

SecAgent 是什么？解决了什么问题？

🚨

安全告警智能研判

告警进来，AI 自动分析真假、溯源、出报告。Kafka/HTTP 双入口，LLM 分流 + 多步推理链。

🔍

漏洞主动扫描

纳管主机批量漏洞扫描，matcher 规则引擎 + Nuclei 双引擎，对话式 + 表单式双入口。

🛡️

自动处置响应

高危事件自动封禁/通知，10套 YAML 剧本 + L1-L5 分级 HITL 审批，企微钉钉通知。

💬

对话式安全助手

自然语言描述扫描需求，AI 理解意图执行。4路路由：scan / project / web / chat。

让 AI 帮安全团队做"告警研判 + 漏洞扫描 + 响应处置"的闭环

三、功能演示脚本

11 个模块，逐个演示，约 30 分钟

模块 1：登录与主题切换

浏览器打开 → 登录页 → 进入大屏（2 min）
功能点
  • JWT 认证登录（admin / analyst / responder / viewer 四种角色）
  • RBAC 权限控制：不同角色看到不同菜单（responder 能看到审批，viewer 只读）
  • 四套 UI 主题：亮色 / 暗色 / 亮侧栏+暗内容 / 暗侧栏+亮内容，一键切换
  • 演示数据注入按钮（管理员可见）
演示操作
  1. 打开 http://localhost:5173 → 看到登录页
  2. 输入 admin / demo123 → 登录成功，进入态势感知大屏
  3. 右上角主题切换下拉框 → 依次切换 4 套主题
  4. 观察左侧导航菜单的权限差异

模块 2：态势感知大屏

首页 Dashboard（3 min）
功能点
  • 4 个统计卡片：总事件数 / 真阳性 / 待审批 / 平均处理时长
  • 饼图：事件结论分布（真阳性 / 假阳性 / 未知 / 忽略）
  • 柱状图：事件定级分布（严重 → 提示，带颜色映射）
  • 趋势折线图：24h 事件趋势，支持时间窗口切换
  • SSE 实时刷新：无需手动刷新，新事件自动更新图表

技术亮点：数据源走 ES 聚合查询；实时通道 SSE（Redis pub/sub 跨 worker fan-out）；中文标签自动映射

模块 3：事件队列与推理链详情

事件队列 → 点击某条 → 事件详情（4 min）
功能点
  • 事件列表：ID / 源 / 定级 / 结论 / 置信度 / 状态 / 时间，支持筛选和分页
  • 推理链 Timeline：每一步 AI 推理过程可视化
  • entry → orchestrator（分流决策）→ cti_analyst（威胁情报查询）→ investigator（综合研判）→ aggregator（汇聚）→ 最终结论
  • IOC 提取结果：IP、域名、Hash、URL（自动脱敏）
  • 审批状态 + SSE 实时追加 trace 步骤

技术亮点：推理链每步带耗时、节点名、输入输出摘要；trace 存 ES 按 event_id 聚合；SSE 实时推送状态变更

模块 4：HITL 分级审批流程

审批列表 → 点击审批 → 批准/拒绝（3 min）
级别触发条件所需票数
L1低风险自动审批（封禁本机 IP）0（自动）
L2自动审批（通知运维群）0（自动）
L3限制单台主机脚本下发1 人 quorum
L4多机协同封锁2 人 quorum
L5跨域防火墙变更2 人 + 风控会签

技术亮点：Redis Lua 原子 quorum · Celery 超时兜底 · 企微/钉钉 webhook 通知 · 审批后自动 execute/reject

模块 5：主机纳管 — Agent 注册与部署

主机纳管 → 创建注册令牌 → 安装 Agent → 主机上线（4 min）
功能点
  • 主机组管理：创建/删除分组，主机可按组归类
  • 注册令牌：生成一次性 enroll token，带 TTL 过期
  • 三种安装方式：一键复制安装命令 / 下载安装脚本 / 手动下载二进制
  • 主机列表：主机名/组/OS/Agent版本/在线状态/最后心跳
  • Agent 生命周期：在线 → 离线 → 已下线（decommission）→ 永久删除（含 token 吊销）
  • Agent 远程升级：服务端推送新版本 → Agent 自升级

注册流程：Agent 启动 → 读取 enroll_token → POST /enroll (Ed25519 key exchange) → 获取 agent_id + token + server_public_key → 保存 config.json → 建立 WebSocket 长连接 → 开始心跳

模块 6：扫描任务 — 表单式 & 对话式

扫描任务 → 表单创建 / 对话式创建（5 min）
表单式创建
  • 目标选择器：按主机组 / 按主机名 / 按 IP-CIDR
  • 扫描模块：系统漏洞 (sys_vuln) / 安全基线 (baseline)
  • 双引擎：matcher（CVE 精确匹配）+ Nuclei（YAML 模板）
  • 资源限制：CPU/内存上限，定时调度 cron
对话式创建（ChatScan）
  • 豆包风格对话界面，自然语言描述 → AI 解析为结构化 ScanIntent → 确认卡片
  • 多轮对话细化需求，多会话管理，模型动态切换
# 示例对话
用户：扫描 web-server 组的所有主机，检查系统漏洞
AI：  好的，将对 web-server 组（3台主机）执行系统漏洞扫描，Nuclei 引擎。
        [确认卡片：目标/模块/引擎] [执行扫描]
用户：加上基线检查
AI：  已更新：扫描模块 = 系统漏洞 + 安全基线。
        [执行扫描]

模块 7：扫描监控 — SSE 实时进度

扫描监控页（3 min）
功能点
  • 实时进度条：总体进度 + 当前主机/模块
  • SSE 事件流：scan_start / host_start / finding / host_done / task_progress / task_done
  • 发现列表：CVE ID / 名称 / 严重等级 / 所在主机 / 检测时间
  • AI 严重等级评估：AI 二次评估修正误报等级
  • 取消扫描 + HTML 报告导出

底层链路：服务端创建任务 → Redis Stream 入队 → TaskWorker → WS 下发 scan_command → Agent 执行 → 实时回传 → ES 写入 → SSE 推前端

模块 8：漏洞清单与扫描报告

漏洞清单 → 筛选 → 批量操作 / 扫描报告（3 min）
漏洞清单
  • 全量漏洞：CVE / 名称 / 严重等级 / AI 评估等级 / 状态(open/fixed/accepted) / 主机 / 类别
  • 筛选（按等级/状态）+ 批量标记操作 + AI 误报过滤标记
扫描报告
  • 按 task_id 查询：统计摘要 + 等级分布 + 主机汇总 + 详细发现列表 + 修复建议
  • HTML 一键导出：包含统计图表 + 详细列表的完整报告，邮件外发无依赖

模块 9：规则管理

规则管理页（2 min）
功能点
  • 规则列表：名称 / CVE / 分类(sys_vuln/baseline) / 严重等级 / 修复建议
  • 筛选搜索 + 规则详情 + 离线导入 ZIP 规则包
  • 规则签名分发：Ed25519 签名 → WS 下发 → Agent 验证 → 热加载
  • 版本管理：规则版本号追踪

分发链路：服务端上传 → 版本号+1 → WS 下发 rule_sync (Ed25519签名) → Agent 验签 → 下载 ZIP → 热加载

模块 10：模型管理

模型管理页（2 min）
功能点
  • 多模型支持：OpenAI / Claude / vLLM / 本地模型
  • 模型 CRUD：添加 / 编辑 / 删除 / 启用 / 禁用
  • 默认模型切换 + 参数配置（temperature / max_tokens / base_url）
  • API Key 安全：列表不返回明文，仅显示是否已配置
  • 预置 4 家模型服务商参数，开箱即用

ModelAdapter 统一抽象层 — 支持 structured output (JSON Schema)，切换模型不影响业务代码，DB 持久化（llm_models 表）

四、模拟外部告警进入（curl 演示）

用 curl 推送告警，模拟 Kafka/HTTP 外部告警进入，配合前端 SSE 实时观察

获取 Token

# 登录获取 JWT token
$ export TOKEN=$(curl -s -X POST http://localhost:8000/login \
    -H "Content-Type: application/json" \
    -d '{"username":"admin","password":"demo123"}' | jq -r '.access_token')

注入演示种子数据

# 一键注入 5 条样例事件（含 L3 待审批）
$ curl -s -X POST http://localhost:8000/api/v1/demo/seed \
    -H "Authorization: Bearer $TOKEN" | jq
→ {"status":"ok","injected":5}

提交自定义告警

# 蜜罐告警 — 会命中 honeypot 快捷通道
$ curl -s -X POST "http://localhost:8000/api/v1/events" \
    -H "Authorization: Bearer $TOKEN" \
    -H "Content-Type: application/json" \
    -d '{
  "sanitized_text": "Honeypot captured whoami && id from 203.0.113.99 on eth0",
  "iocs": {"ip": ["203.0.113.99"], "command": ["whoami","id"]},
  "source": "honeypot"
}' | jq
→ {"event_id":"...","status":"processing"}

# CVE 漏洞利用尝试
$ curl -s -X POST "http://localhost:8000/api/v1/events" \
    -H "Authorization: Bearer $TOKEN" \
    -H "Content-Type: application/json" \
    -d '{
  "sanitized_text": "CVE-2024-5678 exploit: Log4j JNDI injection from 10.0.1.200 to C2 198.51.100.50",
  "iocs": {"ip": ["10.0.1.200","198.51.100.50"], "cve": ["CVE-2024-5678"]},
  "source": "ids"
}' | jq

# 同步模式 — ?sync=true 等待管线完全跑完，实时看推理结果
$ curl -s -X POST "http://localhost:8000/api/v1/events?sync=true" \
    -H "Authorization: Bearer $TOKEN" \
    -H "Content-Type: application/json" \
    -d '{
  "sanitized_text": "Lateral movement via RDP from workstation 10.0.1.100 to DC 10.0.1.10",
  "iocs": {"ip": ["10.0.1.100","10.0.1.10"]},
  "source": "edr"
}' | jq

# 查询推理链
$ curl -s "http://localhost:8000/api/v1/events/<event_id>/trace" \
    -H "Authorization: Bearer $TOKEN" | jq
