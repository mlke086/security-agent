### 修复 1 —— 嵌入维度统一为 1024（P1）

**根因**：`embedding.py` 声明 `EMBEDDING_DIM = 768`，但 BGE-large-zh-v1.5 实际输出 **1024 维**，Milvus 集合也是按 1024 建的。768 这个常量是错的，任何依赖它做补零/校验的地方都会维度不匹配。

**正确改法**（`src/knowledge/graphrag/vector/embedding.py`）：
```python
from functools import lru_cache

EMBEDDING_DIM = 1024  # BGE-large-zh-v1.5 输出 1024 维（此前误写 768）


@lru_cache(maxsize=1)
def _get_model():
    from sentence_transformers import SentenceTransformer
    return SentenceTransformer("BAAI/bge-large-zh-v1.5")


def embed(text: str) -> list[float]:
    vec = _get_model().encode(
        "为这个句子生成表示以用于检索相关文章：" + text,
        normalize_embeddings=True,
    )
    return vec.tolist()


def embed_dim() -> int:
    """供 Milvus 建集合等处引用，避免常量分散。"""
    return EMBEDDING_DIM
```
并把 `milvus_client.py` 的 `VECTOR_DIM = 1024` 改为 `from ...embedding import EMBEDDING_DIM as VECTOR_DIM`，从此单一来源。

**验证**：`python -c "from src.knowledge.graphrag.vector.embedding import embed; print(len(embed('test')))"` 应输出 `1024`。

res = await es.search(index=settings.es_index_audit,

### 修复 7 —— MemoryManager.cleanup() 真实实现（P2）

**根因**：现扫描从未写入的 `event:ts:*` key，且不清理 Milvus/Neo4j，TTL 形同虚设。

**正确改法**：写入证据时同步登记时间戳索引，cleanup 按该索引真实删除三处数据：
```python
# store_evidence 内追加：
await self._redis.zadd("evidence:ts", {doc_id: int(time.time())})

# cleanup 改为：
async def cleanup(self, max_age_hours: int = 24) -> int:
    cutoff = int(time.time()) - max_age_hours * 3600
    stale = await self._redis.zrangebyscore("evidence:ts", 0, cutoff)
    for doc_id in stale:
        await self._delete_from_graph(doc_id)   # Neo4j: MATCH (ev:Evidence {doc_id}) DETACH DELETE ev
        self._milvus.delete(doc_id)             # Milvus: delete by doc_id
        await self._redis.zrem("evidence:ts", doc_id)
    return len(stale)
```
TTL 与文档 10.3.2 对齐为 **24h**（当前 `_DEFAULT_TTL_HOURS=72` 也应改 24）。

**验证**：造一条过期证据，调 cleanup 后 Milvus/Neo4j 中对应记录消失。

### 修复 8 —— Linter 别名/反射绕过加固（P2）

**根因**：`import os as o; o.system()` 或 `getattr(os,'system')` 可绕过危险调用检测——因为只按字面名匹配。

**正确改法**：在 `_import_check` 建立"别名→真实模块"映射，`_dangerous_call_check` 解析调用时先还原别名；并对 `getattr` 调用直接拦截：
```python
# 收集 import 别名
aliases = {}  # alias -> real module
for node in ast.walk(tree):
    if isinstance(node, ast.Import):
        for a in node.names:
            aliases[a.asname or a.name] = a.name
# 危险调用判断时用 aliases 还原前缀；额外拦截 getattr/__import__
if base_name == "getattr":
    return LinterResult(passed=False, error_type="dangerous_call",
                        error_detail="getattr 反射调用被禁止")
```
更稳妥的做法是**导入白名单已排除 os/subprocess**，所以只要保证别名映射后 `os` 这类不在白名单即被拦，双保险。

**验证**：`import os as o\no.system('id')` 被驳回；`getattr(__builtins__,'eval')` 被驳回。
