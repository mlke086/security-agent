### [P0-GO-01] Ed25519.Verify 对 nil PublicKey panic -> agent 崩溃循环
- **位置**：`agent/internal/crypto/verify.go`（`var PublicKey ed25519.PublicKey` 零值 nil；`Verify()` 调 `ed25519.Verify(PublicKey, ...)` 前无 nil guard）
- **现象**：生产 install 脚本路径 `src/agents/enroll.py:INSTALL_SH` 预注册时写 config.json 含 agent_id+agent_token 但**不含 server_public_key**；agent 因 AgentID 已存在跳过 enroll（main.go:46），`crypto.PublicKey` 保持 nil。第一条带 sig 的 scan_command 命中 `ed25519.Verify(nil, ...)` -> Go 运行时 panic "bad public key length: 0" -> 读循环 goroutine 崩 -> 进程退出 -> systemd Restart=always 崩溃循环。HandleUpgrade/HandleRuleUpdate 都做了 nil 检查（upgrade.go:72,125），唯独 Verify 没有，不一致。
- **核实**：CONFIRMED（读 verify.go Verify() 函数体确认无 nil guard；grep enroll.py 无 server_public_key）
- **V3 对应**：P0-GO-1 标记为 PARTIALLY FIXED/REGRESSED--self-enroll 路径通了，但 install.sh 预注册路径 + nil panic 仍在
