# 安全 AI Agent 项目测试方案

> 编制日期：2026-07-14（v2，含每个 API 接口测试）
> 范围：整个项目--既有告警智能分析系统 + 新增主机漏洞扫描子系统。
> 目标：分层覆盖、**每个 API 接口有测试用例**、CI 门禁化、先测后码。

---

## 1. 目标与原则

- 五层覆盖：单元 / API 接口 / 集成 / 契约 / E2E / 安全。
- 单元测试隔离（mock LLM/WS/ES/Redis/系统采集）；API/集成可连真实中间件（192.168.80.101）或 testcontainers。
- 每个接口至少：正常 200 + 鉴权 401/403 + 校验 422 + 边界/异常。
- CI 门禁：ruff/mypy/pytest/go test/tsc/Playwright 全绿。

## 6. 集成测试

- **纳管+通信闭环**（真实 Redis+ES）：建令牌->enroll->WS 连上->心跳->主机在线。
- **扫描全链路**：建任务->dispatch(mock agent WS)->scan_step/result->collect->aggregate->llm_analysis->report。
- **既有告警链**（test_investigation 已有）：submit sync->pipeline->终态。
- **Agent 集成**（Go mock server）：scan_command->各 step->result(is_final)；rule_update 热加载；断线->队列->重连 replay。

---

## 7. 契约测试

- **WS 协议**：信封 `{v,type,ts,sig,payload}` + 各 type payload 双向一致（后端发/Agent 收、Agent 发/后端 收）。
- **OpenAPI 类型**：response_model -> `gen:types` -> schema.ts -> tsc 校验。
- **Ed25519 签验**：后端 `sign_message`（签 `type|ts|json(payload)`）↔ agent `crypto.Verify`；二进制升级 `sign_bytes(sha256(binary))` ↔ `ed25519.Verify(pub, sha256(binary), sig)`；签名能验、篡改拒。
