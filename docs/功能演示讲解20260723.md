## 一、项目概述（2 分钟）

### 我们做了什么？

**SecAgent** 是一套基于大模型（LLM）的**安全事件智能分析与自动响应平台**，核心能力：

1. 🚨 **安全告警智能研判** — 告警进来，AI 自动分析真假、溯源、出报告
2. 🔍 **漏洞主动扫描** — 纳管主机批量漏洞扫描，Agent 端规则引擎 + Nuclei
3. 🛡️ **自动处置响应** — 高危事件自动封禁/通知，L3+ 走人工审批
4. 💬 **对话式安全助手** — 自然语言描述扫描需求，AI 理解意图执行

一句话：**让 AI 帮安全团队做"告警研判 + 漏洞扫描 + 响应处置"的闭环**。

### 模块 3：事件队列与详情（4 分钟）

**演示路径**：事件队列 → 点击某条 → 事件详情

**事件队列功能点**：
- 事件列表：ID / 源 / 定级 / 结论 / 置信度 / 状态 / 时间
- 筛选：按定级、结论、状态过滤
- 分页 + 实时更新（新事件自动出现在列表顶部）

**事件详情功能点**：
- **基本信息卡片**：事件 ID、来源、定级、文本内容
- **推理链 Timeline**：每一步 AI 推理过程可视化
  - entry → orchestrator（分流决策）→ cti_analyst（威胁情报查询）
  - → investigator（综合研判）→ aggregator（汇聚）→ 最终结论
- **IOC 提取结果**：IP、域名、Hash、URL（自动脱敏）
- **审批状态**：若需人工审批，显示审批进度（quorum 人数）
- **SSE 实时更新**：推理过程中逐步追加 trace 步骤，无需刷新

**技术亮点**：
- 推理链每一步带耗时、节点名、输入输出摘要
- trace 存储在 ES，按 event_id 聚合查询
- 状态变更通过 SSE 实时推送前端

**演示操作**：
1. 点击「事件队列」菜单
2. 找到一条 high/critical 事件，点击进入详情
3. 观察推理链 Timeline，理解 AI 研判过程
4. 注意 IOC 提取与脱敏结果
5. 若事件状态为 pending_approval → 跳到审批模块

### 模块 4：审批流程（3 分钟）

**演示路径**：审批列表 → 点击审批 → 批准/拒绝

**功能点**：
- 审批列表：事件 ID / 等级 / 创建时间 / 所需人数 / 当前票数
- L1/L2：自动批准（无需人工介入）
- L3：需 1 人审批 → quorum 达成自动执行
- L4/L5：需 2 人审批 → 双人 quorum
- 审批通知：企微/钉钉 webhook 推送审批卡
- 超时兜底：Celery 定时任务自动 timeout

**技术亮点**：
- Redis 存储审批状态（Hash + Set + ZSet），Lua 脚本原子 quorum
- pub/sub 通知，HITL 等待方实时收到决策
- 审批后自动触发 execute_response 或 reject_response
- 支持 webhook 通知到企业 IM

**演示操作**：
1. 点击「审批列表」（仅 admin/responder 可见）
2. 查看待审批列表，观察 L3 vs L4 的 required_voters 差异
3. 点击一条 L3 审批，选择「批准」
4. 观察事件状态从 pending_approval → completed
5. 确认处置动作已执行（审计日志可查）

### 模块 5：主机纳管 — Agent 注册与部署（4 分钟）

**演示路径**：主机纳管 → 创建注册令牌 → 安装 Agent → 主机上线

**功能点**：
- **主机组管理**：创建/删除分组，主机可按组归类
- **注册令牌**：生成一次性 enroll token，带 TTL 过期
- **安装方式**：
  - 一键复制安装命令（curl pipe bash）
  - 下载安装脚本
  - 手动下载二进制
- **主机列表**：主机名/组/OS/Agent 版本/在线状态/最后心跳
- **Agent 生命周期**：在线/离线/已下线（decommission）
- **Agent 升级**：服务端推送新版本 → Agent 自升级
- **Agent 删除**：软删除(decommission) 或 永久删除(含 token 吊销)

**注册流程（Agent 侧 Go 实现）**：
```
Agent 启动 → 读取 enroll_token → POST /enroll (Ed25519 key exchange)
→ 获取 agent_id + agent_token + server_public_key → 保存 config.json
→ 建立 WebSocket 长连接 → 开始心跳 → 接收规则/扫描指令
```

**演示操作**：
1. 进入「主机纳管」页面
2. 点击「创建注册令牌」→ 设置过期时间 → 生成令牌
3. 复制安装命令 → 在目标主机执行
4. 观察主机列表中出现新主机，状态为「在线」
5. 点击「创建主机组」→ 将主机拖入分组
6. （可选）点击「升级 Agent」→ 推送新版本

### 模块 6：扫描任务 — 表单式 & 对话式（5 分钟）

**演示路径**：扫描任务 → 表单创建 / 对话式创建

#### 6a. 表单式创建

**功能点**：
- **目标选择器**：支持按主机组、按主机名、按 IP/CIDR 选择
- **扫描模块**：系统漏洞 (sys_vuln) / 安全基线 (baseline)
- **扫描引擎**：
  - `matcher`：Agent 端规则引擎，CVE 精确匹配，轻量快速
  - `nuclei`：调用 projectdiscovery/nuclei CLI，支持 YAML 模板
- **资源限制**：CPU/内存上限，避免扫描影响业务
- **定时调度**：支持 cron 表达式定时执行

#### 6b. 对话式创建（ChatScan）

**功能点**：
- 豆包风格对话界面，自然语言描述扫描需求
- AI 理解意图 → 解析为结构化 ScanIntent → 展示确认卡片
- 支持多轮对话细化需求（"扫描哪些主机？" / "需要基线检查吗？"）
- 多会话管理：创建/切换/删除对话
- 模型切换：每轮对话可切换 LLM 模型

**示例对话**：
```
用户：扫描 web-server 组的所有主机，检查系统漏洞
AI：好的，我将对 web-server 组（3台主机）执行系统漏洞扫描。
    使用 Nuclei 引擎，严重等级不限。
    [确认卡片：目标/模块/引擎] [执行扫描]

用户：加上基线检查
AI：已更新：扫描模块 = 系统漏洞 + 安全基线。
    [更新的确认卡片] [执行扫描]

用户：执行
→ 创建任务成功，跳转到监控页
```

**技术亮点**：
- 意图解析在后端静默执行，前端无需额外点击「解析」
- 对话历史持久化存储
- 确认卡片防误操作

**演示操作**：
1. 进入「扫描任务」页面
2. Tab 1「表单创建」：选择目标组 → 选择模块 → 点击创建
3. Tab 2「对话式」：输入自然语言 → AI 解析意图 → 点击执行
4. 观察任务列表出现新任务

### 模块 7：扫描监控 — 实时进度（3 分钟）

**演示路径**：扫描监控页

**功能点**：
- **实时进度条**：总体进度 + 当前主机/模块
- **SSE 事件流**：每台主机扫描结果实时推送
  - 事件类型：scan_start / host_done / finding / task_done
- **发现列表**：CVE ID / 名称 / 严重等级 / 所在主机 / 检测时间
- **AI 严重等级评估**：AI 二次评估修正误报等级

**Agent 端扫描流程**：
```

→ 通过 WebSocket 下发 scan_command 到 Agent → Agent 执行扫描
→ Agent 实时上报结果 (WebSocket) → TaskWorker 写入 ES
→ SSE 推送到前端监控页
```

**演示操作**：
1. 从扫描任务页点击某任务的「监控」
2. 观察进度条逐步增长
3. 观察实时事件流（host_done → finding 逐条出现）

### 模块 11：AI 对话助手（3 分钟）

**演示路径**：扫描任务的对话式 Tab（或独立对话入口）

**功能点**（4 路意图路由）：
1. **scan**：解析扫描意图 → 结构化 ScanIntent
2. **project**：项目文档问答 → 基于 docs/ 本地检索 + LLM 总结
3. **web**：联网搜索 → DuckDuckGo 搜索最新 CVE/安全新闻 + LLM 总结
4. **chat**：自由聊天

**技术实现**：
```
用户消息 → /api/v1/chat → LLM 意图分类（4路之一）
  ├─ scan → 返回 ScanIntent → 前端展示确认卡片
  ├─ project → DocSearch(docs/) → RAG → 带来源回答
  ├─ web → DDG HTML Search → RAG → 带 URL 来源回答
  └─ chat → 直接 LLM 回答
```

**示例问答**：
```
用户：这个系统是怎么做告警研判的？
AI(project)：系统通过 LangGraph 编排引擎处理告警...
              [来源: docs/项目梳理.md]

用户：最近有什么新的高危 CVE 吗？
AI(web)：根据最新搜索结果...
           CVE-2026-XXXXX ...
           [来源: nvd.nist.gov, ...]

用户：你好，你能做什么？
AI(chat)：我是 SecAgent 助手，可以帮你...
```

**演示操作**：
1. 进入对话界面
2. 问一个项目相关问题 → 观察 project 路由
3. 问一个安全新闻问题 → 观察 web 路由
4. 描述一个扫描需求 → 观察 scan 路由 + 确认卡片

## 四、模拟外部告警进入（curl 演示）

演示时可以直接用 `curl` 推送告警，模拟 Kafka/HTTP 外部告警进入，配合前端 SSE 实时观察。

### 获取 Token

```bash
# 登录获取 JWT token
export TOKEN=$(curl -s -X POST http://localhost:8000/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"demo123"}' | jq -r '.access_token')
```

### 注入演示种子数据

```bash
# 一键注入 5 条样例事件（含 L3 待审批）
curl -s -X POST http://localhost:8000/api/v1/demo/seed \
  -H "Authorization: Bearer $TOKEN" | jq
# → {"status":"ok","injected":5}
```

### 提交自定义告警

```bash
# 蜜罐告警 — 会命中 honeypot 快捷通道
curl -s -X POST "http://localhost:8000/api/v1/events" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "sanitized_text": "Honeypot captured whoami && id from 203.0.113.99 on external interface eth0",
    "iocs": {"ip": ["203.0.113.99"], "command": ["whoami", "id"]},
    "source": "honeypot"
  }' | jq
# → {"event_id":"...","status":"processing"}

# CVE 漏洞利用尝试
curl -s -X POST "http://localhost:8000/api/v1/events" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "sanitized_text": "CVE-2024-5678 exploit detected: Log4j JNDI injection from 10.0.1.200 to external C2 198.51.100.50",
    "iocs": {"ip": ["10.0.1.200", "198.51.100.50"], "cve": ["CVE-2024-5678"]},
    "source": "ids"
  }' | jq

# 同步模式 — ?sync=true 等待管线跑完，实时看推理结果
curl -s -X POST "http://localhost:8000/api/v1/events?sync=true" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "sanitized_text": "Lateral movement via RDP from compromised workstation 10.0.1.100 to DC 10.0.1.10",
    "iocs": {"ip": ["10.0.1.100", "10.0.1.10"]},
    "source": "edr"
  }' | jq

# 查询推理链
curl -s "http://localhost:8000/api/v1/events/<event_id>/trace" \
  -H "Authorization: Bearer $TOKEN" | jq
```

提交后前端 Dashboard 通过 SSE 实时刷新，事件队列出现新记录，点击可看到逐步构建的推理链。
