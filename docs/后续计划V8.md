## 一、阶段 1：消除 P1 的“体验性错误”

### 1.1 F2 对话式聊天持久化合一（4-6h）

- 让 `src/api/routers/chat.py` 在分支 `intent == "scan"` 时，把 user+assistant 双方一次性 append 到 `scan_conversations`；同时把 `ChatSource` 里的 `intent snippet` 落库，UI 刷新后看到的就是当时真实那条回复。  
- 删除 `src/api/routers/chat.py` 中再次序列化/再次调 model 的二级模板；保证 `/chat` 端点一个 user input 仅触发一次 LLM round-trip。  
- `frontend/src/api/client.ts:319-346` `chatAssistant()` 把第二次 `chatConversation()` 调用去掉，改为 fetch + 信任 `/chat` 一次性完成。  
- 测试：新增 `tests/unit/api/test_chat_persistence.py`，断言 `/chat` 在 scan 分支下 append 了正确条数，且不触发额外 LLM 调用（用 `chat_completion` mock 的 call_count）。

### 1.2 F3 Agent `config_update` 生效（4-6h）

- `agent/internal/comm/client.go` heartbeat loop 引入 `c.heartbeatInterval` 字段，并把它与 `time.Ticker.Reset()` 绑定；`config_update` 时调用 `c.heartbeatTicker.Reset(newDuration)`，不要再启动 goroutine。  
- `agent/internal/protection/protection.go` 把 `SampleInterval` 与 `CPUPollTimeout` 拆开，前者是采样窗口（实测 250ms-1s 即可），后者只用作 `gopsutil` 的 timeout；避免 1s timeout 套在 5s sample 之上互相零除。  
- 保留 `resource.Monitor` 但只作为节流统计使用；移除 `e.Monitor.IsThrottling()` 在 throttle 流程中对 `time.Sleep(5 * time.Second)` 的无中断等待，改用 `select { case <-ctx.Done() ; case <-time.After(...) }`。  
- 测试：在 `agent/internal/comm/client_test.go` 中新增 `TestApplyConfigResetsHeartbeatTicker`，mock time。

### 1.3 F5 跨 worker 投递幂等（2-4h）

- 给 `scan_cancel` / `agent_upgrade` 加 command id：服务端分配 `cmd_id` 写 Redis；Agent 在 `update_ack` / `send_to_agent` 的 ACK 回传中带 `cmd_id`；服务端以 `cmd_id` 去重，超时重试 1 次后报 409 `command_already_handled`。  
- 同时实现：F2 已经在 `/chat` 端点里持久化时返回 `cmd_id`，聊天流也能幂等。

### 3.3 SP4 主机组删除清理（1-2h）

- 明确产品语义：拒绝还是有成员；选择前在文档里固化。

## 四、阶段 4：Nacos 生产部署前提

### 4.1 SP6 Nacos 启动顺序与热更新边界（5-8h）

- `src/api/main.py` 启动顺序重构为：  
  1. `load_nacos_settings()` 在建连接池前 await 完成；首次失败仍允许启动但记录 WARN。  
  2. poller 后台任务用 backoff + jitter；首次失败后每 30s 重试；成功后切回 60s。  
- `src/common/config/nacos_loader.py` 记录由 Nacos 注入的 key 集合；Nacos 删 key 时清除 `os.environ` 中的注入键；容器显式 env 永远不被覆盖。  
- `settings.py` 将字段分为两组：  
  - “动态生效”：log_level、debug、api_access_token_expire_minutes 等下次请求生效；  
  - “需要重建”：host/url/password、redis_url、es_hosts、pg 池大小；列入白名单，变动时记录需要重启的 WARN。  
- 测试：用 fixture 启动多个 poller case，避免双 listener 同一 key。
