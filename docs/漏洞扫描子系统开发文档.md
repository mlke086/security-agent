## 1. 概述

在现有 security-agent 系统新增"控制台 -> 主系统 -> 子 Agent"主机漏洞扫描能力。本文档覆盖 8 个模块（纳管 / 通信网关 / Go Agent / 规则同步 / 扫描调度子图 / LLM 分析 / 后端 API / 前端）的详细开发思路，以及分层测试方案与流程。

**MVP 范围**：系统漏洞（包/补丁版本比对 CVE）+ 安全基线（账号/端口/密码策略/文件权限/日志）。中间件/弱口令/数据库规则后续阶段补。

**技术栈**：
- 后端：Python 3.11 + FastAPI + LangGraph + Redis + ES（复用现有）
- Agent：**Go 1.22+**（单二进制，linux/amd64、linux/arm64、windows/amd64）
- 前端：React 18 + antd5 + Vite（复用现有）
- LLM：复用 `ModelAdapter`（DeepSeek）
- 测试：pytest + testcontainers（后端）、Go testing（Agent）、Playwright（前端）

## 3. 数据模型详细设计

### 3.1 Pydantic 模型（`src/agents/models.py`）

```python
class HostStatus(str, Enum):
    ONLINE = "online"; OFFLINE = "offline"; DECOMMISSIONED = "decommissioned"

class Host(BaseModel):
    agent_id: str
    hostname: str
    ip: str
    os: str            # linux/windows
    arch: str          # amd64/arm64
    kernel: str
    status: HostStatus = HostStatus.ONLINE
    agent_version: str
    rule_version: str
    last_heartbeat: str
    group: str | None = None
    owner: str | None = None
    env: str | None = None       # prod/test
    created_at: str

class ScanModule(str, Enum):
    SYS_VULN = "sys_vuln"; BASELINE = "baseline"

class ScanPolicy(BaseModel):
    modules: list[ScanModule] = [ScanModule.SYS_VULN, ScanModule.BASELINE]
    resource_limit: dict = {"cpu_percent": 30, "mem_percent": 30}
    time_window: str | None = None      # cron 或时间窗
    timeout_sec: int = 1800

class ScanTask(BaseModel):
    task_id: str
    source: Literal["dialog", "manual"]
    intent_text: str | None = None
    targets: list[str]                 # agent_id 列表
    policy: ScanPolicy
    rule_version: str
    status: Literal["queued","dispatching","scanning","analyzing","completed","failed"] = "queued"
    stats: dict = {"total":0,"done":0,"failed":0}
    created_at: str
    finished_at: str | None = None
    error: str | None = None

class VulnFinding(BaseModel):
    finding_id: str
    task_id: str
    agent_id: str
    hostname: str
    category: ScanModule
    cve: str | None = None
    name: str
    severity: Literal["critical","high","medium","low","info"]
    ai_severity: str | None = None
    ai_filtered: bool = False           # AI 判误报
    evidence: str
    fix_advice: str | None = None
    status: Literal["open","fixed","accepted"] = "open"
    detected_at: str

class ScanResult(BaseModel):
    task_id: str
    agent_id: str
    hostname: str
    findings: list[VulnFinding]
    batch: int
    is_final: bool
    ts: str

class ScanReport(BaseModel):
    task_id: str
    summary: str
    ai_analysis: str
    stats: dict          # {by_severity:{...}, by_category:{...}, total}
    top_vulns: list[dict]
    recommendations: list[str]
    generated_at: str
```

### 3.2 ES 索引（`src/agents/store.py` 建索引）

| 索引 | `_id` | 关键 mapping |
|---|---|---|
| `vulnscan-hosts` | agent_id | status keyword, group keyword, rule_version keyword, last_heartbeat date |
| `vulnscan-tasks` | task_id | status keyword, source keyword, targets keyword(nested) |
| `vulnscan-results` | auto | task_id keyword, agent_id keyword, is_final boolean |
| `vulnscan-vulns` | finding_id | cve keyword, severity keyword, ai_severity keyword, status keyword, hostname keyword |
| `vulnscan-reports` | task_id | generated_at date |

### 3.3 Redis Key

| Key | 类型 | 用途 | TTL |
|---|---|---|---|
| `enroll:{token}` | Hash | 纳管令牌(group/expires/uses) | expires |
| `agent:online:{agent_id}` | String | 心跳时间戳 | 2×heartbeat |
| `agent:conn:{agent_id}` | String | WS 连接所在 worker_id | 心跳刷新 |
| `agent:cmd:{agent_id}` | Pub/Sub | 跨 worker 下发指令 | - |
| `vulnscan:task:{task_id}` | Pub/Sub | 任务进度 -> SSE | - |
| `vulnscan:collect:{task_id}` | Hash | 待回收 agent + 已回收数 | 任务结束清理 |
| `rules:current_version` | String | 当前规则版本 | - |

## 4. 通信协议详细设计

### 4.1 通道

| 通道 | 协议 | 用途 |
|---|---|---|
| 纳管/文件 | HTTPS REST | install 脚本、二进制、规则包下载 |
| 控制/监控 | **WSS**（Agent 主动连出） | 指令下发、心跳、步骤/结果上报 |
| 前端实时 | SSE（复用） | 任务进度推控制台 |

### 4.2 WebSocket 消息 schema

连接：`wss://console/api/v1/agents/ws?agent_id={id}&token={agent_token}`

```jsonc
// 通用信封
{ "v": 1, "type": "<type>", "ts": "2026-07-14T...Z", "sig": "<签名>", "payload": { ... } }
```

**主系统 -> Agent**：
| type | payload |
|---|---|
| `scan_command` | `{task_id, policy, rule_version, modules[], resource_limit, deadline}` |
| `scan_cancel` | `{task_id}` |
| `rule_update` | `{rule_version, download_url, signature}` |
| `agent_upgrade` | `{version, download_url, signature}` |
| `config_update` | `{heartbeat_interval, resource_limit}` |

**Agent -> 主系统**：
| type | payload |
|---|---|
| `heartbeat` | `{agent_version, rule_version, cpu, mem, hostname, ip}` |
| `task_ack` | `{task_id, accepted, reason?}` |
| `scan_step` | `{task_id, step, status, message}` |
| `scan_result` | `{task_id, batch, findings[], is_final}` |
| `update_ack` | `{kind(rule|agent), version, ok, error?}` |

### 4.3 安全（token + TLS）

- **TLS**：主系统对外 HTTPS/WSS，内网自签 CA；Agent 内置 CA 证书校验（`CAPath`）。
- **Agent 鉴权**：WS 连接带 `agent_token`，网关校验（Redis 查 `agent:token:{agent_id}`）。
- **指令签名**：主系统私钥（`settings.agent_signing_key`）对 `type+ts+payload` 签名；Agent 内置公钥校验，仅校验通过的指令执行（防伪造扫描/升级指令）。
- **结果加密**：WSS 已加密传输；敏感字段（弱口令证据等）Agent 端用主系统公钥再加密一层（可选，MVP 可先依赖 TLS）。
- **令牌轮换**：`config_update` 可下发新 `agent_token`。

## 5. 模块A：主机纳管（`src/agents/enroll.py` + `manager.py`）

### 5.1 开发思路

1. **令牌生成**：`POST /api/v1/agents/enroll-tokens`（admin）-> 生成随机 token -> `enroll:{token}` 存 Redis（group/expires/uses）。
2. **安装脚本**：`GET /api/v1/agents/install?token=&os=` -> 返回 bash 或 PowerShell 脚本（按 User-Agent 判断）。脚本：探测 OS/arch -> 下载二进制 -> 装服务 -> 调 enroll -> 启动。
3. **注册**：`POST /api/v1/agents/enroll` -> 校验 token（uses 倒计/失效）-> 生成 `agent_id`(uuid) + `agent_token`(secrets.token_urlsafe) -> 落 `vulnscan-hosts` + Redis `agent:token:{agent_id}` -> 返回凭证 + ws_url + heartbeat_interval。
4. **状态/心跳**：Agent WS 连上即在线；`heartbeat` 消息更新 `agent:online:{agent_id}` TTL + `vulnscan-hosts.last_heartbeat`；TTL 过期 -> 后台任务标 offline。

### 5.2 关键接口

```python
# enroll.py
async def create_enroll_token(group, ttl_hours, uses) -> str: ...
async def validate_enroll_token(token) -> dict | None: ...   # 返回 group，uses--
async def render_install_script(token, os_type) -> str: ...  # bash/ps1
async def enroll_host(token, hostname, os, arch, ip, kernel) -> dict:  # {agent_id, agent_token, ws_url, heartbeat_interval}

# manager.py
async def register_online(agent_id, worker_id) -> None: ...  # agent:conn + agent:online
async def heartbeat(agent_id, payload) -> None: ...          # 刷新 TTL + ES last_heartbeat
async def mark_offline_expired() -> int: ...                  # 周期任务，TTL 过期 -> offline
async def list_hosts(status_filter) -> list[Host]: ...
```

### 5.3 状态机

```
未纳管 --enroll--> 已注册 --WS连上--> 在线 --心跳超时--> 离线 --重连--> 在线
任意 --DELETE /agents/{id}--> 已下线
```

### 5.4 测试用例（单元）

| 用例 | 验证 |
|---|---|
| 令牌生成/校验/uses 倒计/过期失效 | `create_enroll_token` + `validate_enroll_token` 多次调用/超时 |
| install 脚本按 os 返回 bash/ps1 | `render_install_script("linux")` 含 systemd，`("windows")` 含服务注册 |
| enroll 成功落库 + 返回凭证 | mock store，`enroll_host` -> 校验 ES 写入 + agent_token 返回 |
| enroll 无效 token 拒绝 | `validate_enroll_token` 返回 None -> enroll 抛 401 |
| 心跳刷新 TTL | mock redis，`heartbeat` -> `agent:online` TTL 刷新 |
| 离线检测 | 构造过期 key -> `mark_offline_expired` -> ES status=offline |

测试方式：pytest + `fakeredis`/mock store，不依赖真实 Redis/ES。

## 6. 模块B：通信网关（`src/agents/ws_gateway.py`）

### 6.1 开发思路

- **WS 端点**：FastAPI `WebSocket` `/api/v1/agents/ws`，连接时校验 `agent_id+token`（Redis）-> 注册连接到内存表 `_conns: dict[agent_id, WebSocket]` + Redis `agent:conn:{agent_id}=worker_id`。
- **接收循环**：监听 Agent 消息 -> 按 `type` 分发：`heartbeat`->manager，`scan_step`/`scan_result`->store+pub `vulnscan:task:{task_id}`，`task_ack`/`update_ack`->更新任务/更新状态。
- **下发指令**：`send_to_agent(agent_id, msg)` -> 若在本 worker（`_conns` 命中）直接 `send_json`；否则 publish `agent:cmd:{agent_id}` -> 各 worker 订阅 -> 命中者转发。
- **多 worker**：每 worker 启动时订阅 `agent:cmd:*`（pattern）转发；连接断开 -> 清理 `_conns` + Redis `agent:conn`。

### 6.2 关键接口

```python
class AgentGateway:
    async def connect(self, ws, agent_id) -> None: ...
    async def disconnect(self, agent_id) -> None: ...
    async def handle_message(self, agent_id, msg: dict) -> None: ...
    async def send_to_agent(self, agent_id, msg: dict) -> bool: ...   # 跨worker
    async def broadcast(self, agent_ids: list[str], msg: dict) -> dict: ...  # 返回成功/失败
```

### 6.3 测试用例（单元 + 集成）

| 用例 | 验证 |
|---|---|
| 连接鉴权（有效/无效 token） | 有效 -> 注册 `_conns`；无效 -> close(1008) |
| 消息分发（heartbeat/scan_step/scan_result） | 各 type 调对应 handler + ES/pub-sub |
| 本 worker 下发 | `send_to_agent` 命中 `_conns` -> ws.send_json 被调 |
| 跨 worker 下发 | mock Redis pub/sub -> 另一"worker"收到并转发 |
| 断连清理 | disconnect -> `_conns` + Redis `agent:conn` 清除 |

测试方式：单元用 mock WebSocket + fakeredis；集成用 `testcontainers` 起 Redis + 两个 gateway 实例验证跨 worker。

## 7. 模块C：子 Agent（Go，`agent/`）

### 7.1 启动流程（`cmd/agent/main.go`）

1. 加载配置（`/etc/secagent/config.json` 或 `%ProgramData%\secagent\config.json`）：`console_url, ca_path, agent_id?, agent_token?`。
2. 若无 `agent_id` -> 调 `POST /enroll`（带 enrollment token）-> 拿凭证写回配置。
3. 启动 `comm.Client`（WS 连接 + 心跳 goroutine + 重连 backoff）。
4. 启动命令处理循环：收到 `scan_command` -> 起扫描 goroutine；`rule_update`->updater 热加载；`agent_upgrade`->updater 自升级。
5. 信号处理：优雅关闭（完成当前步骤、断连缓存）。

### 7.2 通信模块（`internal/comm`）

- `Client.Connect()`：WSS 拨号（带 CA 校验 + `agent_id+token` query），重连指数退避（1s->30s 上限）。
- 心跳 goroutine：每 `heartbeat_interval` 发 `heartbeat`（含 cpu/mem/rule_version）。
- `Send(msg)`：发消息；`onMessage` 回调分发到 handler。
- 断线缓存：未发出的 `scan_result` 写本地 SQLite/文件队列，重连续传。

### 7.3 扫描引擎（`internal/scan`）

**Collector（采集，平台分支）**：
- `sys_vuln` 采集：
  - Linux：`dpkg-query -W`（deb系）/ `rpm -qa`（rpm系）取包+版本；`uname -r` 内核；`/etc/os-release` 发行版。
  - Windows：`Get-HotFix`（补丁 KB）、`Get-Package`/注册表已装程序、`[System.Environment]::OSVersion`。
- `baseline` 采集：
  - 账号：`/etc/passwd`、`/etc/shadow` 锁定态、Windows 本地用户。
  - 端口：`ss -tlnp` / `netstat`。
  - 密码策略：`/etc/login.defs`、PAM 配置；Windows `secedit`。
  - 文件权限：关键路径（`/etc/passwd`、`/etc/ssh/sshd_config`、`.ssh`）。
  - 日志：`rsyslog`/`journald` 配置；Windows 事件日志策略。

**Rules（规则比对）**：
- 规则包格式（签名 JSON）：
  ```jsonc
  { "version":"2026.07.14", "rules":[
    {"id":"CVE-2024-1234","category":"sys_vuln","cve":"CVE-2024-1234","name":"...","severity":"high",
     "check":{"type":"package_version","name":"openssh","op":"<","value":"9.0"},"fix":"升级 openssh>=9.0"},
    {"id":"BL-001","category":"baseline","name":"SSH root 登录未禁用","severity":"high",
     "check":{"type":"config_check","file":"/etc/ssh/sshd_config","pattern":"^PermitRootLogin","expect":"no"}}
  ], "signature":"..." }
  ```
- `Matcher.Match(collected, rules) -> []Finding`：按 `check.type` 分派（`package_version`/`config_check`/`kernel_version`）。

**Engine（编排）**：
```
scan_command -> task_ack
  -> step:collect_packages(running->done) -> step:match_cve(running->done) -> findings
  -> step:baseline_check(running->done) -> findings
  -> resource 检查点（每模块间）
  -> scan_result(batch, is_final=true)
```
每步发 `scan_step`，结果分批发 `scan_result`。

### 7.4 资源限流（`internal/resource`）

- 启动时读 `resource_limit`（cpu_percent/mem_percent）。
- 采样 CPU/Mem（Linux `/proc/stat`、Windows `GetSystemTimes`/性能计数器）。
- 超阈值 -> `time.Sleep` 退让/暂停扫描（发 `scan_step` status=running message=throttled）。
- `time_window` 外（业务高峰）-> 拒绝/延迟扫描（`task_ack accepted=false reason=time_window`）。

### 7.5 规则热更新与自升级（`internal/updater`）

- `rule_update` -> 下载规则包 -> 校验签名（内置公钥）-> 原子替换内存规则 + 落盘 -> `update_ack ok`（不重启）。
- `agent_upgrade` -> 守护进程下载新版二进制 -> 校验签名 -> 替换 -> 重启服务（systemd/服务管理器）-> 新版连回 -> `update_ack`。

### 7.6 跨平台构建

- `Makefile`：`GOOS=linux GOARCH=amd64|arm64`、`GOOS=windows GOARCH=amd64` -> `dist/secagent-{os}-{arch}`。
- `packaging/install.sh`（bash）/ `install.ps1`：下载二进制 + 装服务（systemd unit / `sc create`）+ 写配置 + enroll + 启动。

### 7.7 测试用例（Go testing）

| 模块 | 用例 |
|---|---|
| Collector | 用 fixture（样本 `/etc/passwd`、`dpkg-query` 输出）测试解析；Windows 用 mock 输出 |
| Matcher | 表驱动：`package_version`/`config_check`/`kernel_version` 各 正例/反例/边界 |
| Resource | 注入 fake CPU 采样 -> 超阈值触发退让 |
| Report | findings 序列化/分批/`is_final` 正确 |
| Comm | 用 `gorilla/websocket` 起本地 mock server：心跳、scan_command->scan_step->scan_result、重连 |
| Updater | 假规则包（正确/错误签名）-> 热加载成功/拒绝 |
| 集成 | mock WS server 模拟主系统，跑完整 scan_command 流程 |

测试方式：Go `testing` + table-driven + testdata fixtures；mock WS server 用 `httptest`+`websocket.Upgrader`。

## 8. 模块D：规则库在线同步（`src/agents/rules_sync.py`）

### 8.1 开发思路

- **数据源**：NVD API（CVE，按时间增量拉取）+ CNNVD（国标，可配置 RSS/API）。MVP 先 NVD。
- **同步任务**：Celery beat 或 APScheduler 周期任务（每日）-> 拉取新增 CVE -> 转换为规则（`check` 多为 `package_version`/`kernel_version`，需 CVE→受影响包/版本映射，可用 NVD CPE）。
- **签名与版本**：生成规则包 JSON -> 主系统私钥签名 -> 存 `rules:pack:{version}`（ES 或文件）+ 更新 `rules:current_version`。
- **下发**：Agent 心跳上报 `rule_version` -> 落后则主系统发 `rule_update`（带 download_url + signature）。
- **手动触发**：`POST /api/v1/agents/rules/sync`（admin）立即同步；`GET /rules` 查版本历史。

### 8.2 关键接口

```python
async def sync_rules(source="nvd", since=None) -> str:  # 返回新 version
```

## 9. 模块E：扫描任务调度子图（`src/orchestration/subgraphs/vulnscan/`）

### 9.1 子图

```
parse_intent(仅dialog) -> dispatch -> collect -> aggregate -> llm_analysis -> report -> END
```

### 9.2 各节点开发思路

- **parse_intent**：仅 `source=dialog` 时调 `ModelAdapter.chat_completion(schema=ScanIntent)`，把 `intent_text` 解析为 `{targets(主机组/IP/host), modules, resource_limit, schedule}`；`source=manual` 跳过。ScanIntent Pydantic 校验。
- **dispatch**：创建 `ScanTask`（ES，status=dispatching）-> 解析 targets 为 agent_id 列表 -> `gateway.broadcast(agent_ids, scan_command)` -> 记 `vulnscan:collect:{task_id}` 待回收数 -> status=scanning。
- **collect**：异步等所有目标 `scan_result(is_final=true)` 或超时。实现：订阅 `vulnscan:task:{task_id}` 收 `scan_result` 事件，计数达待回收数 -> 进入 aggregate；超时 -> 带部分结果继续。
- **aggregate**：查 ES `vulnscan-results`（by task_id）-> 合并 findings -> 去重（同 host+check/CVE）-> 标准化 -> 落 `vulnscan-vulns`。
- **llm_analysis**：见模块F。
- **report**：构建 `ScanReport`（stats + top_vulns + recommendations）-> 落 ES -> 发 `vulnscan:task:{task_id}` 完成 -> SSE 推前端。

### 9.3 状态与超时

- 每节点带超时（dispatch 60s、collect = policy.timeout_sec、analysis 300s）。
- 失败 -> task.status=failed + error -> SSE。

### 9.4 测试用例

| 用例 | 验证 |
|---|---|
| parse_intent 解析 | mock ModelAdapter -> ScanIntent 字段正确；manual 跳过 |
| dispatch 下发 | mock gateway.broadcast -> scan_command 发到所有 targets；待回收数正确 |
| collect 收齐/超时 | 模拟 N 个 scan_result -> 收齐进 aggregate；超时带部分结果 |
| aggregate 去重 | 多 host/同 CVE 合并；同 host 同 check 去重 |
| report 生成 | stats 正确 + ES 落库 + pub 完成事件 |

测试方式：mock ModelAdapter + mock gateway + memory/fake ES store；pytest-asyncio。

## 10. 模块F：LLM 分析（`vulnscan/nodes.py` 的 llm_analysis）

### 10.1 开发思路

- 输入：聚合后 findings（去重）+ 主机上下文（资产等级/环境）。
- Prompt（复用 `ModelAdapter`）：
  ```
  你是漏洞研判专家。对以下扫描发现做：1) 误报过滤（标 ai_filtered=true 并给理由）
  2) 风险分级（结合 CVSS + 资产等级，输出 ai_severity）3) 修复建议（补丁/缓解/回滚）
  主机上下文: {hosts_context}
  发现列表: {findings}
  返回 JSON: {analyzed: [{finding_id, ai_severity, ai_filtered, reason, fix_advice}]}
  ```
- 输出 Pydantic `AnalyzedFinding` 列表 -> 回填 `vulnscan-vulns`（ai_severity/ai_filtered/fix_advice）。
- 分批：findings 多时分批调 LLM（每批 ≤50）合并。

### 10.2 测试用例

| 用例 | 验证 |
|---|---|
| 误报过滤 | mock LLM 返回 ai_filtered -> 回填正确 |
| 风险分级 | 资产核心+中危 -> ai_severity=high |
| 修复建议 | 回填 fix_advice |
| 分批 | 100 条 findings -> 2 批 LLM 调用 -> 合并 |
| LLM 失败降级 | mock 抛错 -> 跳过 AI 分析，原 severity 保留，不阻断报告 |

测试方式：mock `ModelAdapter`，pytest。

## 13. 安全设计详细

- **TLS**：主系统 `uvicorn --ssl-certfile --ssl-keyfile`；自签 CA 由 `scripts/gen_ca.py` 生成；Agent 内置 `ca.pem`。
- **令牌**：enrollment token（短期/限次，Redis）、agent_token（长期，Redis `agent:token:{agent_id}`，可轮换）。
- **指令签名**：Ed25519，主系统私钥签 `type+ts+payload`，Agent 内置公钥验；`scan_command/rule_update/agent_upgrade` 必验。
- **审计**：纳管、令牌、任务、下发、规则同步、Agent 升级、结果接收全部 `AuditLogger` -> ES，满足 6 个月留存。
- **数据最小化**：Agent 仅上报 findings（CVE/等级/证据摘要），不上传原始配置文件。
- **应急脚本（后续）**：白名单账号 + 签名 + 超时 + 审计，MVP 不启用。
