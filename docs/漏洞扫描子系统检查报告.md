## 1. Go Agent 问题（最严重，需重点返工）

> 来自 Go 深审代理（已跑 go build/vet/test 全挂 + 逐文件审查）。

### 1.1 无法构建（Critical）

- **[Critical] `agent/go.mod` + 4 个 `.go` 文件有 UTF-8 BOM**（`agent/internal/{comm/client.go, config/config.go, enroll/enroll.go, version/version.go}`）。`go build/vet/test` 全报 `unexpected input character '﻿'`，整个 agent 编译不了。与之前 playbook YAML 同一 BOM 老坑。
  - 修：去 5 文件 BOM（re-save UTF-8 no BOM）。
- **[Critical] `agent/internal/queue/queue.go:15`** import `_ "modernc.org/sqlite"` 但 go.mod 只声明 `gorilla/websocket`，无 go.sum；该库最新版需 go1.25，当前 go1.24.4、go.mod 声明 go1.22。
  - 修：补依赖并 pin go1.22 兼容版，或换纯 Go/CGo-free SQLite，或去掉该包。
- **[High] `enroll.go:11`、`engine.go:7`** 未用的 `"time"` import -> 编译错。
- **[Medium] 缺失**：无 `Makefile`（文档 §7.6 要求）、无 `packaging/agent.xml`、**无任何 `*_test.go`**（文档 §7.7 要求大量表驱动测试）、`rules/` 空目录。

### 1.2 扫描永远零发现（Critical，核心功能失效）

- **[Critical] `engine.go:107,128`** 匹配以 `len(cmd.Rules) > 0` 为门，但服务端 `scan_command` 不发 `rules` 字段（文档 payload 是 `{task_id, policy, rule_version, modules, resource_limit, deadline}`）；规则经 `updater`->`scan.LoadRules` 热加载，engine 却从不调 `scan.GetRules()`。**结果：每次扫描匹配全跳过，零发现**。
  - 修：engine 用 `scan.GetRules()`（fallback 到 inline），按 `rule_version` 选规则。
- **[High] `scan/rules.go:21-30`** `LoadRules` 把规则包（对象 `{version,rules,signature}`）反序列化进 `[]RuleDef`（切片）-> 必然失败，规则集永远空。
  - 修：反序列化进 `struct{Version string; Rules []RuleDef; Signature string}`，load `.Rules`。
- **[High] `matcher.go:110-166`** `config_check` 恒误报：collector 没采文件**内容**（只采权限），baseline 正则永远匹配不到，且 `!exists` 就出 finding -> **每条 baseline 规则都误报**。
  - 修：采文件内容并应用正则；仅"配置存在且错误"或"必存在却缺失"才出 finding。
- **[High] `matcher.go:189-213`** `compareVersions` 用 `strconv.Atoi` 切片比较，遇 `1.2.3-1.el9`/`1:1.2.3`/`1.2.3~rc1` 等非数字后缀 -> Atoi 失败按 0 -> 大量误报/漏报。
  - 修：用 `golang.org/x/mod/semver` 或 dpkg-version 风格比较（处理 epoch/revision/distro 后缀）。
- **[Medium] `engine.go:114,133,143`** findings 发两遍（增量批 + 末尾全量重发）-> 后端收双份。
- **[Medium] `engine.go:168`** `hostname="unknown"`（engine 没取主机名，`getHostname()` 在 comm 包未导出）-> 发现无法归属主机。
- **[Medium] engine 无 deadline/超时/资源检查点**（文档 §7.3/§7.4 要求模块间 throttle 检查点 + deadline）。
- **[Low] `collector.go:339-344`** `getUID` 断言 `Uid()` 方法，但 Linux `*syscall.Stat_t.Uid` 是字段 -> uid 恒 "0"。
- **[Medium] Windows baseline 大量未实现**：`collectPasswordPolicy` 返 nil（缺 secedit）、`collectLogConfig` 空（缺事件日志）、`collectPackagesWindows` 用弃用的 `wmic qfe` 非 `Get-HotFix`、`net user` 解析不过滤表头行 -> 表头变假账号。
- **[Low] 缺采集**：`/etc/os-release`（distro）、`/etc/shadow` 锁定态（文档 §7.3 要求）。

### 1.3 安全漏洞（RCE 级，Critical）

- **[Critical] `crypto/verify.go` 整包死代码**：`Verify`/`SetPublicKey` 从未被调用，`PublicKey` 从未设置。`comm/client.go:135-167` `handleMessage` 只解析 `type/payload`，丢弃 `sig/ts`，所有服务端指令（scan_command/rule_update/agent_upgrade/scan_cancel/config_update）**不验签**。与文档 §13（Ed25519 必验）矛盾。
  - 修：启动调 `SetPublicKey`，`handleMessage` 解析完整信封 `{v,type,ts,sig,payload}`，`crypto.Verify` 通过才分发。
- **[Critical] `updater/upgrade.go:62`** 升级签名只比对 sha256，且 `req.Signature==""` 时**跳过校验** -> 攻击者注入空签名 `agent_upgrade` 推任意二进制 = **RCE**。
  - 修：强制非空 Ed25519 签名，用 `crypto.PublicKey` 验下载字节，空/无效拒绝。
- **[High] `updater/upgrade.go:99-128`** 规则包 `Signature` 字段被忽略，不验签直接 `LoadRules`（且反序列化失败）。
  - 修：验包签名再加载。

### 1.4 连不上主系统（Critical/High）

- **[Critical] `comm/client.go:61-64`** 用 `websocket.DefaultDialer` 无 `TLSClientConfig`，`cfg.CAPath` 存了从不用 -> 自签 CA 不被信任，WSS 拨号失败。
  - 修：读 `CAPath` 进 `*x509.CertPool`，设 `dialer.TLSClientConfig.RootCAs`。
- **[High] `comm/client.go:42-49`** 强制 `wss://`，但 install 脚本写 `console_url=http://192.168.80.101:8000` -> wss 打 http 端口 -> TLS 握手失败 -> 无限重连。
  - 修：`http->ws`、`https->wss` 映射。

### 1.5 其他 Go 问题

- **[High] `comm/client.go:87/106-133`** 每次重连新起心跳 goroutine，旧的从不关（`c.done` 从不 close）-> goroutine 泄漏。
- **[High] `comm/client.go:169-178`** 断线时 `send()` 静默返回，`internal/queue` 从未被 comm 调用 -> 断线时 scan_step/result/task_ack **全丢**（与文档"断线缓存续传"矛盾）。
- **[Medium] `comm/client.go:107`** 心跳 ticker 一次构造，`config_update` 改 `HeartbeatSec` 不重置 ticker -> 实时改心跳间隔无效。
- **[Medium] `comm/client.go:117-128`** 心跳硬编码 `cpu:0,mem:0,rule_version:""`，缺 `ip`；`rule_version` 热加载后不更新 -> 后端无法知 agent 跑哪版规则。
- **[Medium] `comm/client.go:162-163`** `scan_cancel` 只 log，engine 无取消机制 -> 运行中扫描无法取消。
- **[Critical] `resource/monitor.go:78-102`** 采样错：内存用 `runtime.MemStats`（Go 进程堆，非系统内存）、CPU 用 `NumGoroutine/(GOMAXPROCS*4)`（非 CPU 占用）。且 **resource 包从未被 engine 调用** -> 限流死代码。
  - 修：Linux `/proc/stat`+`/proc/meminfo`、Windows `GetSystemTimes`/`GlobalMemoryStatusEx`；engine 模块间调 `IsThrottling()`。
- **[High] `updater/upgrade.go:79-88`** Windows 运行中二进制不能 rename -> 升级静默失败。
- **[High] `updater/upgrade.go:88` + `main.go:101-105`** `os.Exit(0)` 在发成功 ack 之前 -> 后端收不到升级成功 ack。
- **[Medium] `main.go:24-30`** 配置缺失/损坏时重新 `config.Load`（仍返 nil）再解引用 `cfg.AgentID` -> nil panic。
- **[Medium] `enroll.go:77-88`** `getLocalIP()` 返主机名非 IP；`getKernelVersion()` 返字面 "Linux"/"Windows" 非 `uname -r` -> 后端 Host 记录垃圾。
- **[Low] `enroll.go:50-54`** `http.Post` 无超时无 TLS -> 服务端不可达时永久挂起。
- **[Low] `crypto/verify.go:56-67`** 规范化经 `map[string]interface{}`（数字变 float64），脆弱；`sort` 是 no-op 死代码。
- **[Medium] `main.go:119,130`** `config_update` ack 用 `kind="config"`，但文档 `update_ack.kind` 只有 `rule|agent`。
- **[High] `packaging/install.ps1:53`** `Out-File -Encoding utf8` 写 config 带 BOM -> `config.Load` 失败。
- **[Medium] `install.ps1:30`** arch 检测两分支都返 amd64。
- **[Medium] `install.sh:22`/`install.ps1:6,17`** 写 `console_url=http://...` 与 agent 强制 wss 矛盾。
- **[Low] `install.sh:23`** `armv7l->arm` 映射到未构建的 GOARCH -> 下载 404。
- **[Low] `agent.service:9,13`** 设 `CONFIG_PATH` 但 `config.DefaultConfigPath()` 不读该 env（死 env）。
