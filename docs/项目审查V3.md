## 一、P0 - 阻断级（核心功能跑不通 / 严重安全）

### [P0-VS-1] vulnscan WS 网关认证永远失败，agent 无法连接
- **位置**：`src/agents/ws_gateway.py:35-39`
- **现象**：`authenticate` 读 Redis `agent:token:{agent_id}`，但全代码库**无任何地方 SET 该 key**（仅 `manager.py:74` 的 delete 碰过）。`register_enroll_token` 把 token 的 SHA256 哈希存进 PG `agent_tokens`；`validate_agent_token`(`enroll.py:155`, PG 校验) 定义了却**从未被调用**。
- **后果**：每个 agent 的 WS 握手认证都返回 False -> **没有任何 agent 能连上控制台**。vulnscan 整条扫描链路从连接环节就断。
- **修复方向**：`authenticate` 改为调用 `validate_agent_token(agent_id, token)`（PG 校验哈希），或注册时同步写 Redis。
- **核实**：已当场 grep 确认（仅 read + delete，无 set；validate_agent_token 仅定义处出现）。**CONFIRMED**。

### [P0-VS-2] vulnscan 任务 task_id 不匹配，用户拿到的 id 查不到结果
- **位置**：`src/api/routers/vulnscan.py:44-46` vs `src/orchestration/subgraphs/vulnscan/graph.py:34` + `nodes.py:25`
- **现象**：路由生成 `task_id` 返回给用户，但 `asyncio.create_task(run_vulnscan(...))` 不传 task_id；`run_vulnscan` 无该参数；`_default_state` 自己生成新 uuid。用户的 task_id 只用于审计日志，真正的 ES 文档/SSE 通道用的是另一个 uuid。
- **后果**：`GET /tasks/{id}`、`/tasks/{id}/stream`、`/reports/{id}` 对用户全部 404/空。
- **修复方向**：`run_vulnscan` 加 `task_id` 参数透传，`_default_state` 接收而非自生成。
- **核实**：已当场读三处源码确认。**CONFIRMED**。

### [P0-GO-1] Go Agent 服务器公钥从未下发，scan_command 致崩溃循环或静默丢弃
- **位置**：`agent/internal/crypto/verify.go:24`（`PublicKey` 包变量）+ `agent/cmd/agent/main.go`（无 `SetPublicKey` 调用）+ `agent/internal/config/config.go`（无公钥字段）
- **现象**：`SetPublicKey` 只在测试里调用；Config 无服务器公钥字段；安装脚本不写。
- **后果**：
  - 无 `sig` 的 `scan_command` -> "missing signature" -> 静默丢弃（零发现）
  - 带 `sig` 的 `scan_command` -> `ed25519.Verify(PublicKey=nil,...)` -> **Go 运行时 panic**（bad public key length）-> 读循环 goroutine 崩 -> agent 进程崩溃，systemd `Restart=always` 变崩溃循环
  - `agent_upgrade` 永远拒绝（"server public key not configured"）
- **修复方向**：Config 加 `server_public_key` 字段；启动时 `crypto.SetPublicKey`；后端 `signing.get_public_key_hex()` 通过 `/ca` 或 enroll 响应下发。
- **核实**：子代理 grep 确认 SetPublicKey 仅测试调用。**CONFIRMED**。

### [P0-SEC-1]（降级说明）直接 RCE 已堵，但指令验签面不全
- 旧担心"签名不验=直接 RCE"在当前代码**基本被推翻**：Go 采集器只跑硬编码命令（dpkg/rpm/uname 等），规则无可执行 payload；二进制升级有 Ed25519 闸门。**无直接 shell-exec RCE**。
- 但仍存 P0 级可用性/完整性风险（见 P0-GO-1）+ P1 级 SSRF/完整性（P1-GO-2）。
